# NaijaHomes Property Management Project

A fictional case study created as part of my Business Analysis portfolio. This project simulates the documentation a Business Analyst would produce when supporting the planning and operational rollout of a property management platform for the Nigerian real estate market.

## Solution

The response to this project was not a single isolated document — it was a connected set of BA artifacts, each addressing a different phase of the project lifecycle: defining **what** needs to be built and **why**, and defining **how** the business will run it once built.

## Document architecture

**01 — Business Requirements Document (BRD)**
- Business context, objectives, and **7** measurable KPIs tied to stakeholder-agreed success metrics
- **51** functional requirements across 9 modules (authentication, property/unit management, tenancy, rent payments, service charges, maintenance, communications, reporting, admin), prioritized using **MoSCoW**
- Non-functional requirements, assumptions, constraints, and a **7**-item risk register with likelihood/impact/mitigation
- Regulatory mapping to Nigeria-specific frameworks (NDPA 2023, CBN guidelines, NIBSS, State Tenancy Law)
- Acceptance criteria defined as the formal gate for go-live sign-off

**02 — Standard Operating Procedure (SOP)**
- **7** end-to-end operational procedures (property onboarding, tenant onboarding, rent collection, arrears escalation, service charge billing, maintenance handling, tenant offboarding), each broken into numbered steps with a named responsible role
- Escalation matrix mapping issue types to first/second/third points of contact with target resolution times
- QA checklists and process KPIs to keep execution consistent across staff and onboarded partners
- Records retention schedule aligned to the regulatory requirements set out in the BRD
